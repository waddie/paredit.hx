## Missing – not yet implemented

Neither `nvim-paredit` nor `paredit.hx` implements these. They exist only in `paredit.el`.

### `paredit-splice-sexp-killing-forward` (`M-↑`)

Splice the enclosing form, discarding everything **after** the cursor. The element
under the cursor (and anything before it) is kept inline in the parent.

```common-lisp
(foo (bar |baz quux) zot)
           ↓
(foo bar zot)
```

Effectively: delete-to-end-of-form then splice, fused into one operation.

### `paredit-splice-sexp-killing-backward` (`M-↓`)

Splice the enclosing form, discarding everything **before** the cursor. The element
under the cursor (and anything after it) is kept inline in the parent.

```common-lisp
(foo (bar baz |quux) zot)
           ↓
(foo quux zot)
```

The mirror of the above.

### `paredit-convolute-sexp` (`M-?`)

Pull the **head** of the enclosing form up one level, nesting the grandparent’s head
inside the current form instead. Inverts one layer of nesting.

```common-lisp
(let ((x 1))
  (foo |bar baz))
           ↓
(foo (let ((x 1))
       bar)
     baz)
```

The grandparent’s head (`let ((x 1))`) moves inside the current form; the current
form’s head (`foo`) rises to replace the grandparent’s position. Elements after the
cursor (`baz`) remain siblings of the promoted head.

## Intentionally excluded – Helix already handles it

### Navigation

| `paredit.el`                                 | `nvim-paredit`                   | Helix equivalent                         |
| -------------------------------------------- | -------------------------------- | ---------------------------------------- |
| `paredit-forward` / `paredit-backward`       | `move_to_next/prev_element_head` | `w`/`b`/`e`; `A-o` + nav                 |
|                                              | `move_to_next/prev_element_tail` | `e`/`:move_prev_word_end`                |
| `paredit-forward-down`                       |                                  | `A-o` (expand selection into child node) |
| `paredit-backward-up` / `paredit-forward-up` | `move_to_parent_form_start/end`  | `A-o` (expand selection to parent)       |
| `paredit-backward-down`                      |                                  | `A-i` (shrink selection into child)      |
|                                              | `move_to_top_level_form_head`    | `gg` / `[f` / `]f`                       |

| ------------------------------ | ---------------------------------------- |

| `select_in_top_level_form`     | `mi(` at top level                       |

### Deletion

| `nvim-paredit`             | Helix equivalent                      |
| -------------------------- | ------------------------------------- |
| `delete_form`              | select form (`ma(`) then `d`          |
| `delete_in_form`           | select form contents (`mi(`) then `d` |
| `delete_top_level_form`    | select top-level form then `d`        |
| `delete_in_top_level_form` | select top-level contents then `d`    |
| `delete_element`           | `A-o` to select element, then `d`     |
